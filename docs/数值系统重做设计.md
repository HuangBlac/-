## 一、新旧数值对照

### 原有系统 → 新系统

| 原有属性 | 新属性 | 说明 |
|----------|--------|------|
| 知识 | EDU知识 | 知识广度，理解深度 |
| 灵感 | INT直觉 | 创新思维，应对挑战 |
| 技能树 | SEN感知 | 操作精度，发现细节 |
| - | STR耐力 | 长时间活动，承受压力 |
| 人际关系 | SOC社交 | 交流，获取资源 |

### 属性说明

```
INT直觉 (Intelligence)
  - 范围: 0-100
  - 影响: 创新idea提出、解决复杂问题、应对突发挑战
  - 科研: idea创新值加成、实验成功率加成
  - 事件: 能力检定成功率
  - 特殊: 高直觉（>=70）时，可以将大成功要求的骰点变成小于等于5。

SEN感知 (Sensation)
  - 范围: 0-100
  - 影响: 阅读论文效率、发现细节、操作精度、察觉异常
  - 科研: 文献阅读速度、实验操作精度
  - 事件: 发现隐藏线索
  - 特殊：随机事件触发事件，理智损失更多

EDU知识 (Education)
  - 范围: 0-100
  - 影响: 知识广度、理解深度、研究方法选择
  - 科研: 研究效率、论文质量
  - 事件: 理解神话知识
  - 特殊：高知识时，可以将大成功要求的骰点变成小于等于10。

STR耐力 (Stamina)
  - 范围: 0-100
  - 影响: 长时间实验、田野调查、承受压力、身体抗性
  - 科研: 田野调查效率、实验持续时间
  - 特殊: 高耐力时，进行活动可能不消耗行动点
          ("你感觉你还可以继续做，从而不扣除本次行动值")

SOC社交 (Social)
  - 范围: 0-100
  - 影响: 与导师/同门/审稿人交流、获取资源、合作研究
  - 科研: 导师指导效果、审稿人态度
  - 事件: 社交事件成功率
  - 特殊: 高社交时，你可以与导师/同门/审稿人获得的收获会更高。
```

## 二、数值计算公式

### 2.1 科研相关计算

```
文献阅读:相关值SEN，INT
首先进行一个感知判定1d100
1d100=1：大成功，
你感觉这篇文章的知识直接流入了你的脑海”

进度=max(70,i*2)，i为当前阅读进度

若成功，则正常按下面的方法增加进度
成功进度 = (10 + SEN感知/10) × (1 + EDU知识/100)
              基础10 + 感知/10 + 知识加成

若判定失败：
你感觉这篇文章有些晦涩难懂，进度不良
进度 = 10 * （1 + EDU知识/100）

若大失败：
你感觉你过往所认知到的关于（你所选取的科研方向，例如外神）的知识全都是错误的，搞错了搞错了搞错了（大失败提示）
进度 = min(30, i/2),i为当前进度

文献阅读达到100%时，获得一个“原始idea”
原始idea包括数值为创新值
此处进行一个灵感判定：
大成功：获得一个创新值为10的原始idea
"你感觉你才思泉涌，想到了一个从未有过的天才想法"
成功：获得按如下公式的创新值
"你冥思苦想，想到了个差不多的点子"
Idea创新值 = 基础创新值 × (1 + INT直觉/50)
            （EDU/200+INT/100）*100%+失败次数/2
失败：未能生成idea，文献阅读进度回到70%-80%
大失败：
原始idea不生成，文献进度清零
你感觉你过往所学习的内容就是一团灰尘
接着，按照概率决定是否接受idea
计算公式为p = 1 - (1 - 0.5)^(创新值/100)
按照p的概率决定是否接受，否则该idea需要经过打磨。

按照创新性决定是否接受实验

实验成功率 = 50% + INT直觉×0.3% + SEN感知×0.2% + EDU知识×0.2%
           - 理智修正(理智<50时-10%, <30时-20%)

论文质量 = (创新值平均×2 + 实验支撑×2 + 理论深度) × (1 + EDU知识/200)
```

### 2.2 行动点消耗

```
标准消耗: 1行动点

STR耐力触发"继续工作":
  - 消耗: 0 (不消耗本次行动点)
  - 触发条件: STR耐力 > 70 且 随机(0,100) < (STR耐力-70)/2
  - 限制: 每局游戏最多触发N次 (防止刷分)

导师安排杂事(低水平导师):
  - 消耗: 1-2行动点(取决于导师类型)
  - 触发条件: 导师水平 < 40
```

### 2.3 事件检定

```
基础成功率 = 50%

能力检定:
  成功 = 基础 + (对应属性 - 要求值)×5% + INT直觉×2%

理智修正:
  理智<50: ×0.8
  理智<30: ×0.6
  理智<10: ×0.4
```

## 三、期刊系统

### 3.1 期刊分类

```
┌─────────────────────────────────────────────────────────────┐
│                     期刊类型                                  │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  外神期刊 (顶级)                                             │
│  - 要求: 创新值≥9, 理论深度≥4, 实验支撑≥3                    │
│  - 毕业效力: 1篇 = 直接毕业                                  │
│  - 导师限制: 只有导师能力值≥95才能投递                        │
│  - 示例: 《Cosmic Annals》《Nature》等                       │
│    (需5篇顶刊才能继续跟着奈亚导师深造)                         │
│                                                              │
│  综合期刊 (顶级)                                             │
│  - 要求: 创新值≥7, 理论深度≥3, 实验支撑≥2                    │
│  - 毕业效力: 2篇 = 等同1篇外神期刊                           │
│  - 示例: 《Science》《Nature》等                             │
│    (对标现实顶级期刊)                                        │
│                                                              │
│  子刊                                                        │
│  - 要求: 创新值≥6, 理论深度≥2, 实验支撑≥2                   │
│  - 毕业效力: 2篇 = 1篇综合期刊                              │
│  - 示例: 《XX科学》《XX研究》等                              │
│                                                              │
│  学科期刊 (普通)                                             │
│  - 要求: 创新值≥5                                           │
│  - 毕业效力: 1篇                                            │
│  - 示例: 《克苏鲁研究》《神话学刊》等                         │
│                                                              │
│  会议论文 (特殊)                                             │
│  - 要求: 创新值≥8, 仅需1个成熟结果                          │
│  - 特点: 创新要求高, 但只需1个结果                          │
│  - 毕业效力: 3篇 = 1篇学科期刊                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 期刊数据结构

```python
class Journal:
    """期刊"""
    journal_id: str
    name: str
    type: JournalType  # 外神/综合/子刊/学科/会议
    requirements: dict  # {"创新值": 9, "理论深度": 4, "实验支撑": 3}
    graduation_value: int  # 毕业效力值
    is_restricted: bool  # 是否需要特殊条件(如导师能力≥95)

class JournalType(Enum):
    OUTER_GOD = "外神期刊"      # 顶级，需导师95+
    COMPREHENSIVE = "综合期刊"   # Science/Nature级
    SUB_JOURNAL = "子刊"        # 顶级期刊的子刊
    DISCIPLINE = "学科期刊"     # 普通期刊
    CONFERENCE = "会议论文"     # 特殊类型
```

### 3.3 投稿流程

```
撰写初稿完成
    ↓
选择投稿类型:
  [1] 外神期刊 (要求: 创新9+, 理论4+, 实验3+)
  [2] 综合期刊 (要求: 创新7+, 理论3+, 实验2+)
  [3] 子刊 (要求: 创新6+, 理论2+, 实验2+)
  [4] 学科期刊 (要求: 创新5+)
  [5] 会议论文 (要求: 创新8+, 仅需1个结果)
    ↓
计算投稿成功率
    ↓
判定结果:
  - 直接接受 (低概率)
  - 小修
  - 大修
  - 拒绝
```

## 五、导师系统联动

```
导师水平 < 40:
  - 安排横向项目
  - 强制消耗行动点: -1~2
  - 事件: "导师让你去财务处报销发票"

导师水平 40-80:
  - 无特殊影响

导师水平 81-95:
  - 安排小导师带
  - 小导师能力 = 导师能力 - 20

导师水平 96-99:
  - 非人类超凡生物
  - 毕业要求: 5篇外神期刊才能继续深造

导师水平 = 99 (奈亚提亚普):
  - 特殊导师(隐藏)
  - 研究方向锁定: 旧日支配者
  - 毕业要求: 5篇外神期刊
```
