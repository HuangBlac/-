# 克研模拟器 - Bug 清单与修复计划

## 短期目标：跑通现有流程

### 🔴 严重 Bug（会导致崩溃）

#### 1. Player 缺少 skills 属性
**位置**：
- `game_engine.py:126` - `for skill, level in self.player.skills.items()`
- `research.py:330-331` - `skill = random.choice(list(self.player.skills.keys()))`
- `research.py:339-340` - 同上

**问题**：Player 类根本没有定义 `skills` 属性，但多处代码在使用它

**影响**：evaluate_idea 时会直接 AttributeError 崩溃

**修复方案**：
- 方案A：在 Player.__init__ 中添加 `self.skills = {}`，但这个属性没有实际用途
- 方案B（推荐）：删除所有 skills 相关代码，改用新五维属性（INT/SEN/EDU/STR/SOC）

---

#### 2. 旧属性系统残留
**位置**：
- `graduation.py:55` - `self.player.knowledge // 10`
- `graduation.py:96` - `self.player.inspiration * 0.01`
- `research.py:456` - `self.player.knowledge // 10`
- `action_handlers.py:361` - `self.player.inspiration += inspiration_gain`

**问题**：代码中混用了旧属性（knowledge/inspiration）和新属性（INT/SEN/EDU/STR/SOC）

**影响**：虽然 Player 类保留了 knowledge/inspiration 属性（初始值10），但这些属性不会随游戏进程增长，导致数值计算不合理

**修复方案**：
- 统一改用新五维属性
- `knowledge` → `EDU`（知识）
- `inspiration` → `INT`（直觉/灵感）

---

### 🟡 中等 Bug（影响体验但不崩溃）

#### 3. 事件系统只影响旧属性
**位置**：`event_system.py:164-168`

**问题**：事件效果只修改 knowledge/inspiration，不修改新五维属性

**影响**：玩家触发事件后，属性变化不明显，事件系统形同虚设

**修复方案**：
- 在 apply_event_effect 中添加对 INT/SEN/EDU/STR/SOC 的修改
- 或者在事件 JSON 中直接定义新属性的变化

---

#### 4. Debug 行动暴露给玩家
**位置**：`game_engine.py:272-275`

**问题**：A/B/M 三个 debug 行动直接显示在正式行动列表中

**影响**：破坏游戏平衡和沉浸感

**修复方案**：
- 添加 `debug_mode` 配置项
- 只在 debug_mode=True 时显示这些行动

---

#### 5. 研究方向选择逻辑不完善
**位置**：`course.py:309-342`

**问题**：
- 只根据选修课名称关键词判断方向，容易误判
- 如果所有计数都是0，随机选择方向，玩家无法控制

**影响**：玩家可能被分配到不想要的研究方向

**修复方案**：
- 在选课时明确提示每门课对应的研究方向
- 或者让玩家在研一结束时手动选择研究方向

---

#### 6. 毕业论文开题流程不完整
**位置**：`action_handlers.py:388-391`

**问题**：开题时只提示"请先输入毕业论文题目"，但没有实际的输入接口

**影响**：玩家无法开始毕业论文

**修复方案**：
- 添加题目输入逻辑
- 或者自动生成题目（基于研究方向和已发表论文）

---

### 🟢 轻微问题（不影响游戏但需要优化）

#### 7. 结局文本过于简陋
**位置**：`game_state.py:40-60`

**问题**：延期毕业只打印"请继续努力"就结束了

**影响**：缺少情感冲击，玩家体验不完整

**修复方案**：
- 为每种结局编写完整的叙事文本
- 参考 `docs/narrative_content.md` 中的结局叙事

---

#### 8. 课程数据文件缺失检查
**位置**：`course.py:28-80`

**问题**：如果 `data/courses.json` 不存在，会直接崩溃

**影响**：首次运行或文件丢失时无法启动

**修复方案**：
- 添加文件存在性检查
- 提供默认课程数据作为 fallback

---

#### 9. 事件触发条件过于严格
**位置**：`event_system.py:71-108`

**问题**：很多事件的触发条件互相冲突，导致某些事件永远不会触发

**影响**：内容利用率低，玩家重复看到相同事件

**修复方案**：
- 重新设计事件触发权重系统
- 记录已触发事件，降低重复概率

---

#### 10. 导师压力事件触发率不合理
**位置**：`advisor.py:268-294`

**问题**：
- 奈亚导师的爆压事件直接理智回满，过于强大
- 普通导师的爆压事件触发后异变+0.05，但没有明确说明这是什么

**影响**：游戏平衡性问题

**修复方案**：
- 调整奈亚导师的效果（理智回满但异变大幅增加）
- 为爆压事件添加更丰富的叙事文本

---

## 额外发现的设计问题

### 1. 游戏节奏问题
- 研一阶段（课程）和研二研三阶段（科研）割裂感强
- 缺少过渡和铺垫，玩家突然从"上课"切换到"做科研"

### 2. 数值系统混乱
- 新旧属性并存，没有统一的设计思路
- 很多数值（如 knowledge、inspiration）初始值10，但从不增长

### 3. 缺少反馈机制
- 玩家做出选择后，看不到明确的属性变化
- 研究进度、论文质量等核心数值不透明

### 4. 事件系统利用率低
- 1140行事件数据，但触发条件过于严格
- 很多精心设计的事件玩家可能永远看不到

---

## 建议的重构方向（中长期）

1. **统一属性系统**：彻底移除 knowledge/inspiration，只用五维属性
2. **重新设计研一阶段**：让课程选择对后续科研有明确影响
3. **增加可视化反馈**：在控制台UI中显示属性变化、进度条
4. **优化事件系统**：降低触发门槛，增加事件多样性
5. **完善叙事内容**：将 `narrative_content.md` 中的片段整合到游戏中
